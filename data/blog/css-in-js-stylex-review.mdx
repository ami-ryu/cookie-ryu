# Stylex

- 새로운 프로젝트를 Next.js 14 (app router) 로 셋팅 예정인데, Emotion 보다 더 괜찮은 CSS-in-JS 도구가 있을까 찾아보다가, 2023년동안 Github에 올라온 프로젝트들 기준, 가장 많이 사용된 CSS-in-JS 도구가 Tailwind 랑 메타에서 만든 Stylex 이길래, Stylex 를 좀 살펴봤다.
- 1줄요약: 다른거 쓰자.

### 장점

- 제로 런타임
- defineVars() 로 전역에서 사용해야하는 값을 쉽게 import 가능
- defineVars 로 선언하면 타입 추론이 잘 된다.
- 스타일 그룹화가 쉽다
- React 엘리먼트 단위로만 styled component 를 생성할 수 있는 타 라이브러리와 달리, stylex 에서는 유사한 스타일끼리 서로 묶어서 선언하고 사용 가능

### 단점

- 아직 v1.0 이 안됨
- SWC 지원을 안해서 Babel로만 컴파일 가능
- 이로인해 Next.js Server Action 과 next/font 사용 불가
- 기존 컴포넌트의 스타일에 오버라이딩 하고 싶을 때 처리가 번거롭다.
- 부모 요소에서 자식 요소 스타일링이 안된다.
